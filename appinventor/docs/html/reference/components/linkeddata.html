ObjectType

Object Type specifies a Uniform Resource Identifier (URI) for a type used to identify objects that should appear in the list picker.

For example, http://xmlns.com/foaf/0.1/Person (foaf:Person), is a commonly used class for identifying people. Specifying it here would generate a list of people from the Endpoint URL.

PropertyURI

If the list picker is placed within a Linked Data Form, the Property URI specifies the relationship between the object being built by the form and the item selected in the list picker.

For example, an application for disaster reporting may query a hierarchy of disaster types and present those types using the Linked Data List Picker.

RelationToObject

RelationToObjectType specifies the relationship between objects presented by the list picker and the Object Type. This field defaults to rdf:type, but other useful values may include skos:narrower or schema:additionalType. The correct property to use will be dependent on the data available in Endpoint URL.

Returns the URI of the userâs selection.
Shape
Specifies the shape of the LinkedDataListPicker. The valid values for this property are 0 (default), 1 (rounded), 2 (rectangle), and 3 (oval). The Shape will not be visible if an Image is used.
ShowFeedback
Specifies if a visual feedback should be shown when a LinkedDataListPicker with an assigned Image is pressed.
SubjectIdentifier

If the list picker is placed within a Linked Data Form, the Subject Identifier value indicates whether the selected item in the list picker should be used as an input when the Linked Data component generates a new subject identifier for the thing described by the form.

Reasoner

The Reasoner component derives statements from the contents of a LinkedData component.

Properties

Model
Specifies the base model (A-Box + T-Box) for reasoning.
RulesEngine
Specifies the base semantics that should be used for reasoning.
RulesFile
An optional file containing custom rules to be used during reasoning. The rules are applied in addition to any reasoning applied by the choice of RulesEngine.

Events

ErrorOccurred(message)
Runs when the reasoner encounters an error during reasoning.
ReasoningComplete()
Runs when the reasoner has been prepared and any forward-chaining rules have finished.

Methods

AddRulesFromRuleset(ruleset)
Adds additional rules to an existing ruleset.
GetLangStatements(subject,predicate,object,lang)
Method for GetLangStatements
GetStatements(subject,predicate,object)
Get statements as a list of triples from the knowledge base. Each argument can either be false or a string. False is treated as a wildcard. Strings are interpreted as URIs.
Query(query)
Synchronously evaluate a SPARQL query over the knowledge base. The return type depends on the type of query run. For SELECT queries, the return value is a JSON-like dictionary containing results in the SPARQL 1.1 Query Result format.
RulesFromRuleset(ruleset)
Populate the rule-based reasoner with the given ruleset of custom rules.
RulesetToString()
Return the loaded rules (if any) as a string.
Run()
Runs the reasoner in forward chaining model to derive conclusions. On success, the ReasoningComplete event will run. ErrorOccurred event will run if reasoning fails.
